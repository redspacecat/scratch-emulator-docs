Scratch Emulator documentation Basic Notes: - kind of like lisp syntax - the command and its arguments are surrounded by parentheses - even when there is no arguments, command is still surrounded by parentheses - arguments follow command after a space, separated by spaces, on one line - one command a line unless nested - jump and jump_if jump to line number in machine code, not hll - strings are surrounded by double quotes - use // at the begining of the line for comments - example: if (= (pick_random (-10 10)) 5) (say_for_secs (join "Hello person #" (pick_random 1 1000)) 2) endif SPECIAL NOTES ON KEYWORDS There are a few special keywords that are NOT functions that still do stuff. Do not surround them with parentheses Indentation is not necessary You can use "true" and "false" as constants without quotes or parentheses. The condition for if and repeat_until go after the keyword and a space, for example: if (= (+ 1 1) 2) (say_for_secs "Yay" 2) endif To make a repeat forever, you would do: repeat_until false (move_steps (pick_random 1 5)) endloop if (CONDITION) - starts an if statement. If must be followed by either else or endif else - if the condition for the if is not true endif - ends a if. You MUST have this after either an if or the if's else repeat_until (CONDITION) - repeats until specified condition is true endloop - ends a repeat_until. You MUST have this at the end of a loop fast - start a run without screen refresh section of code. Do not put a forever loop in here. endfast - ends fast. Put this when you want to go back to normal speed View the examples in the project for examples of how to use keywords. ---------------------------------------- Motion ---------------------------------------- move_steps () turn_right () turn_left () go_to_random go_to_mouse go_to_xy () () glide_secs_to_mouse () glide_secs_to_random () glide_secs_to_xy () () () point_in_direction () point_towards_random point_towards_mouse change_x () set_x () change_y () set_y () if_on_edge_bounce set_rotation_style () -
